# Project instructions

## 基本方針

- このリポジトリは Japan Remote Guide の本番ウェブサイトです。
- 依頼に必要な範囲だけを最小限に変更してください。
- 明示的な依頼がない限り、既存のデザイン、URL、ページ構成、内部リンク、SEO、アクセシビリティ、Cloudflare Pagesとの互換性を維持してください。
- 無関係なリファクタリング、依存関係の更新、ファイル名変更、URL変更を行わないでください。

## 作業前の確認

ファイルを変更する前に、必ず以下を確認してください。

1. 対象ファイルとその周辺の実装
2. `package.json` に定義されたスクリプト（存在しない場合は、その事実を確認すること）
3. `.github/workflows/` 内のGitHub Actions
4. GitHub Actionsが実際に使用している検証コマンド
5. 既存のHTML、内部リンク、アセットの記述方法

このリポジトリでは、GitHub Actionsの検証で少なくとも以下が実行されます。

```bash
npx --yes html-validate@latest "public/**/*.html"
node scripts/check-internal-links.mjs
```

## HTML編集時の必須ルール

- 新規作成または編集したHTMLは、既存のhtml-validateルールに適合させてください。
- HTML本文や属性値に表示用の `&` を記述する場合は、必要に応じて `&amp;` としてください。
- 既存のHTMLエンティティを二重にエスケープしないでください。
- `<title>` はリポジトリの検証ルールで許可される文字数以内にしてください。
- 現在の検証ルールが70文字以内の場合、必ず70文字以内にしてください。
- titleの文字数を作業完了前に実際に確認してください。
- `aria-label` は有効かつ必要なHTML要素にだけ使用してください。
- 不要なARIA属性を追加しないでください。
- ARIAエラーをルール無効化によって隠さないでください。
- 見出し階層、ランドマーク、リンク、ボタンを意味的に正しいHTMLで実装してください。
- 重複したID、壊れた内部リンク、存在しない画像やアセットへの参照を追加しないでください。

## 作業後の必須検証

HTMLを新規作成または編集した場合は、作業完了前に必ず以下を実行してください。

```bash
npx --yes html-validate@latest "public/**/*.html"
node scripts/check-internal-links.mjs
```

HTMLを変更していない場合でも、変更内容がサイト構成、リンク、アセット参照、Cloudflare Pagesの挙動に影響する可能性があるときは、該当する検証コマンドを実行してください。
